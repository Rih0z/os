# Phase 2: Input/Output

## 初心者向け：I/Oって何？

### I/Oの例え話

```
┌─────────────────────────────────────────────────────────────┐
│  I/O = Input/Output = 入力と出力                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  例えば、レストランでのやり取り：                           │
│                                                             │
│  お客さん（CPU）    ウェイター（ドライバ）    厨房（デバイス）│
│  ┌─────┐           ┌─────┐              ┌─────┐           │
│  │     │  注文する  │     │   伝票を渡す  │     │           │
│  │ 客  │ ────────► │ウェイター│ ────────► │ 厨房 │           │
│  │     │           │     │              │     │           │
│  │     │ ◄──────── │     │ ◄─────────── │     │           │
│  │     │  料理を受け取る  │   料理を受け取る│     │           │
│  └─────┘           └─────┘              └─────┘           │
│                                                             │
│  OSでも同じ！                                               │
│  - CPUは直接デバイスを操作しない                            │
│  - ドライバが仲介役                                         │
│  - ドライバがデバイスとやり取りする                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### I/Oデバイスの種類

| 種類 | 特徴 | 例 |
|------|------|-----|
| **ブロックデバイス** | ブロック単位、ランダムアクセス | HDD, SSD, USB |
| **キャラクタデバイス** | 1バイトずつ、ストリーム | キーボード, マウス |
| **ネットワークデバイス** | パケット単位 | NIC |

### 用語の読み方と意味

| 用語 | 読み方 | 意味 |
|------|--------|------|
| **I/O** | アイオー | Input/Output（入出力） |
| **デバイス** | でばいす | ハードウェア機器 |
| **ドライバ** | どらいば | デバイスを制御するソフトウェア |
| **割り込み** | わりこみ | デバイスからの通知 |
| **ポーリング** | ぽーりんぐ | 定期的にチェック |
| **DMA** | ディーエムエー | Direct Memory Access（自動転送） |

## MINIX 3のドライバアーキテクチャ

### マイクロカーネルの特徴

```
┌─────────────────────────────────────────────────────────────┐
│  MINIX 3 = ドライバもユーザー空間で動く                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  モノリシックカーネル（Linux等）                            │
│  → ドライバがクラッシュ = カーネル全体がクラッシュ          │
│                                                             │
│  マイクロカーネル（MINIX 3）                                │
│  → ドライバがクラッシュ = そのドライバだけ再起動            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### MINIX 3のドライバ一覧

| ディレクトリ | 内容 |
|-------------|------|
| `drivers/at_wini/` | ATA/IDE ディスク |
| `drivers/tty/` | 端末（キーボード・画面） |
| `drivers/rs232/` | シリアルポート |
| `drivers/ethernet/` | イーサネット |

## デッドロック

### デッドロックとは？

```
┌─────────────────────────────────────────────────────────────┐
│  デッドロック = お互いに待ち合って進まない状態              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  プロセスA: リソースXを持ってYを待つ                        │
│  プロセスB: リソースYを持ってXを待つ                        │
│  → 誰も動けない！                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### デッドロックの4つの条件

1. **相互排除** - リソースは同時に1つだけ
2. **保持と待機** - 保持しながら待つ
3. **剥奪不可** - 強制的に取り上げられない
4. **循環待機** - 待ち関係がサイクル

### デッドロック対策

| アプローチ | 説明 |
|-----------|------|
| 予防 | 4つの条件のどれかを崩す |
| 回避 | デッドロックが起きない要求だけ許可 |
| 検出と回復 | 検出したら解決する |
| 無視 | 稀だから無視（Linux, Windows） |
